Vấn đề

Kiến thức fix bug bay mất sau mỗi sprint

Dev đã fix bug rồi commit, tuần sau đồng nghiệp gặp lại lỗi tương tự, mất 2-4 giờ debug lại từ đầu. KB rải rác Slack/Confluence/head. Search không hiểu tiếng Việt bỏ dấu.

❌ Không có OneMCP

  • • Dev A fix bug xong → commit → quên
  • • 3 tuần sau Dev B gặp lại lỗi tương tự
  • • Debug lại từ đầu, mất 2-4 giờ
  • • KB rải rác nhiều nơi, không tìm được
  • • Search không unaccent, thiếu ngữ cảnh

✅ Với OneMCP

  • • Dev A fix → agent tự submit KB có template
  • • KB có structure chuẩn (problem/solution/related)
  • • Dev B paste error → agent search → hit
  • • Apply fix trong 5-10 phút
  • • FTS unaccent-aware + fuzzy title match

Skills vs Artifacts — phân biệt

📘 Skills

Static protocol / guide / SOP — do maintainer viết.

  • • Source: GitLab mono-repo skills-kythuat
  • • Sync tự động (webhook + cron)
  • • Format: manifest.json + SKILL.md
  • • Ví dụ: debug-guide, session-wrapup

📄 Artifacts

User-generated knowledge — dev submit qua MCP hoặc portal.

  • • 3 loại: report · research · kb
  • • Template validate structure
  • • Versioning (append-only)
  • • Review flow: pending → published
  • • Attachments PDF/PNG… ≤100MB
Cài đặt

Kết nối Claude Code với OneMCP

3 phút cài đặt. Yêu cầu: đăng ký username với maintainer trước.

1

Thêm MCP server vào Claude Code

Mở file ~/.claude/settings.json và thêm:

{
  "mcpServers": {
    "onemcp": {
      "type": "http",
      "url": "https://onemcp.local/api/mcp",
      "headers": { "X-Onemcp-User": "your-username" }
    }
  }
}
2

Verify kết nối

Trong Claude Code chat, hỏi:

# hỏi agent
liệt kê skills khả dụng trong onemcp

# agent sẽ tự gọi list_skills và trả về danh sách skills của phòng
3

Cài PreCompact hook (khuyến nghị)

Nhắc bạn submit KB trước khi session compact:

# download hook script
mkdir -p ~/.claude/hooks
curl -o ~/.claude/hooks/onemcp-wrapup-check.py \
  https://onemcp.local/hooks/onemcp-wrapup-check.py
chmod +x ~/.claude/hooks/onemcp-wrapup-check.py

# env vars (~/.bashrc hoặc ~/.zshrc)
export ONEMCP_BASE_URL="https://onemcp.local"
export ONEMCP_USER="your-username"

# register hook trong ~/.claude/settings.json → hooks.PreCompact

Default: advisory (chỉ nhắc). ONEMCP_HOOK_STRICT=1 để block compact nếu chưa submit.

Workflow điển hình

Debug bug end-to-end

6 bước từ khi gặp error đến khi KB dùng chung được cho dev khác.

1
Gặp error Paste stacktrace/log vào Claude Code chat, mô tả context (endpoint gọi, khi nào xảy ra).
2
Agent search Agent tự động gọi search với keywords từ error. Nếu hit KB → apply thẳng, skip debug.
3
Load guide nếu cần Nếu chưa có KB, agent gọi load_skill debug-guide để lấy protocol 6-step: stabilize → gather → reproduce → isolate → fix → wrap-up.
4
Fix theo protocol Agent hỗ trợ debug + code. Bạn confirm mỗi bước quan trọng (không apply blind).
5
Wrap-up KB Trước khi close session, agent gọi get_artifact_template kb → điền problem/solution/related → submit_artifact.
✓
Maintainer review → Published Trong ngày, maintainer approve. KB dept-wide visible qua search. Dev khác dùng ngay.

Template KB — ví dụ payload

{
  "type": "kb",
  "title": "Fix: BullMQ worker OOM khi Redis maxmemory-policy=allkeys-lru",
  "slug": "kb-bullmq-oom-redis-eviction",
  "structured": {
    "problem": "Worker BullMQ crash sau 30 phút, log OOM. Redis INFO memory hiện evicted_keys tăng liên tục.",
    "solution": "Redis config maxmemory-policy phải là noeviction cho BullMQ. allkeys-lru evict job data → worker đọc key null → retry loop → OOM. Fix trong docker-compose.yml redis command.",
    "related": "BullMQ docs khuyến nghị. Check /health/redis memory metric để phát hiện sớm."
  },
  "tags": ["bullmq", "redis", "infra"]
}

Search tips

✅ Nên

  • • Dùng error message thật (unaccent auto)
  • • 3-6 từ khoá cốt lõi
  • • Có thể bỏ dấu (huong dan = hướng dẫn)
  • • Filter kind=artifact nếu chỉ tìm KB

❌ Tránh

  • • Câu hỏi dài dạng chat
  • • Từ stop-word không thông tin
  • • Search 1 từ generic (error)
  • • Kỳ vọng semantic (v1 keyword-based, v2 sẽ có)
Governance

Vai trò & Review flow

5 roles, mỗi role kế thừa quyền role trước. Multi-tenant enforce ở service layer — không leak cross-dept.

RoleQuyền chính
viewerĐọc published skills/artifacts (trong dept)
contributor+ submit artifact (pending), sửa artifact của mình (optimistic lock)
maintainer+ approve/reject pending, publish version
dept-admin+ quản lý user role trong dept
super-admin+ cross-dept operations, config hệ thống, admin CIDR
contributor: submit_artifact ──▶ status=pending, v1 │ maintainer review ┌─────┴──────┐ ▼ ▼ published rejected (searchable) (feedback)
FAQ

Câu hỏi thường gặp

Data có leak sang phòng ban khác không? ▾

Không. Mọi query filter department_id theo user identity. Multi-tenant enforce ở service layer.

Nhỡ submit sai thì sao? ▾

Artifact là append-only versioning. Edit tạo version mới, maintainer có thể publish version cũ nếu cần rollback.

MCP dùng được với tool khác ngoài Claude Code? ▾

Được — bất kỳ MCP client nào (Cursor, Continue, custom). JSON-RPC 2.0 chuẩn, chỉ cần config header X-Onemcp-User.

Search không thấy KB tôi mới submit? ▾

KB pending không xuất hiện với user khác. Bạn thấy KB của mình. Đợi maintainer approve để dept-wide visible.

Khi nào có semantic search? ▾

P4.2 roadmap — cần chọn embedding infra (Ollama local? cloud?). Feedback pilot sẽ quyết định ưu tiên.

Backup có tự động không? ▾

Có. Daily pg_dump + MinIO mirror, retention 14 ngày. Off-site mirror (NAS/S3) đang manual — sẽ auto hoá post-v1.

Nếu OneMCP down thì Claude Code còn dùng được không? ▾

Có. MCP tools chỉ là extension. Claude Code core hoạt động bình thường, chỉ không gọi được 7 tools OneMCP cho tới khi service back up.
